# mooresolutions.io

A visually striking, technologically impressive personal website and blog for Daniel Moore.

## Overview

This project is a redesign of Daniel Moore's personal website. The primary goal is to create a fast, lightweight, and accessible platform to showcase projects and share thoughts on technology and programming.

## Features

- **Static Site Generation:** Built into static HTML, CSS, and JavaScript.
- **Markdown-to-HTML:** A directory-based system for blog posts and projects using Markdown files.
- **Frontend Framework:** Powered by **Vue 3** for modular frontend components.
- **Terminal-Inspired Design:** A universal dark theme mimicking a terminal / TUI, inspired by the `oh-my-zsh` **halflife** theme.
- **Fully Responsive:** 100% responsive and optimized for touch devices.
- **Lightweight & Fast:** Designed for performance, SEO, and accessibility.
- **Custom SVG Icons:** Performance-optimized SVG icons generated using PIL (Python Imaging Library).

## About Daniel Moore

Daniel Moore is a web developer (started in 2014) specializing in backend development, with proficiency in frontend and a current focus on native mobile development using Swift and Kotlin.

## Tech Stack

- **Frontend:** Vue 3, HTML, CSS, JavaScript.
- **Styling:** Custom TUI/Terminal-inspired dark theme.
- **Deployment:** GitHub Pages.
- **CI/CD:** GitHub Actions.
- **DNS & Domains:** Namecheap and Cloudflare.

## Project Structure

- `/src`: Vue 3 components and logic.
- `/content/blog`: Markdown files for blog posts.
- `/content/projects`: Markdown files for projects.
- `/scripts`: Utility scripts (e.g., PIL-based SVG generation).

## Content & Demo Files

Both blog posts and project showcase entries are written in **Markdown**. The site includes demo files to showcase formatting and features, including:
- **Embedded Images:** High-performance, optimized images.
- **Links:** Internal and external linking.
- **Styling:** Code blocks, lists, and standard Markdown formatting compatible with the Terminal/TUI theme.

### Demo Content Locations:
- `content/blog/demo-post.md`: A sample blog post demonstrating Markdown features.
- `content/projects/demo-project.md`: A sample project entry with links and visual elements.

## Deployment

Continuous deployment is handled via GitHub Actions, deploying directly to GitHub Pages. DNS management is handled through Cloudflare, with the domain registered at Namecheap.
